User Experience Research

Much of the work that I do as a researcher is protected by non-disclosure agreements. I conducted the entire UX research process on my own website, AutumnAllYear.us to demonstrate the elements in a way that is open and accessible to the public.

The Client:

AutumnAllYear.us - a recipe website

The Ask:

Comprehensive User Experience Research resulting in Recommendations, Prototyping, and Site Testing after change implementation.

Process Overview:

  1. Beginning with a Heuristic Evaluation - a thorough assessment of a product's user interface - I revealed several areas of strength and several opportunities.
  2. I next conducted a Site Inventory and Audit to identify what content exists on the site, what content is working well, and where there is room for improvement.
  3. Next, I constructed a Survey to compile site user metrics, which I used to create User Personas.
  4. With personas created, the next step was to create user stories to help identify what things site visitors come to the site hoping to find. It can be a useful tool for understanding what elements of a site work well, what elements are missing, and where opportunities exist.
  5. Next, I constructed Usability Testing Materials and used them to conduct in-person Usability Tests.
  6. Using the results from the Usability Tests, I constructed a Prototype demonstrating recommended site changes.
  7. With Prototypes created, and with stakeholder buy-in, the next step is to conduct Prototype Usability Testing with representative site visitors.
  8. With the prototype testing complete, I provided a detailed summary of the Research Results and Recommendations and presented them to stakeholders in a recorded Presentation.
  9. Continued research has led to iterative changes.

Process Details:

  1. Heuristic Evaluation

    2. The first step was to conduct a heuristic evaluation using the Nielsen Normal Group's 10 Usability Heuristics for User Interface Design. The results revealed several areas of strength as well as a few elements that needed to be addressed in the next release of the site.

    Open Heuristic Evaluation in Google Docs

  2. Content Inventory and Audit

    3. The second step is to conduct a content inventory and audit. This is a step that can help identify what content exists on the site, what content is working well, and where there is room for improvement.

    Open Heuristic Evaluation in Google Sheets

  3. User Research and Persona Creation

    4. The next step is to conduct user research and use that data to create representative users in the form of Personas.

    User research was conducted via surveys. Surveys were distributed and data collected was compiled to create four representative users.

  4. User Stories

    5. With personas created, the next step is to create user stories. User stories help with identifying what things site visitors come to the site hoping to find. It can be a useful tool in understanding what elements of a site work well, what elements are missing, and what elements may need to be changed or removed.

    Open User Stores in Google Docs

  5. Usability Testing Materials

    6. With the previous steps completed, usability testing materials can be drafted. These materials will be used to conduct live testing with representative site visitors.

    Open Usability Testing Materials in Google Docs

  6. Prototypes

    7. Prototypes are interactive representations of a site that can be utilized to test a site before coding or prior to making changes to the current code. It gives designers and other stakeholders the ability to preview the changes before moving on to coding. Prototypes are also valuable tools for conducting usability testing with representative site visitors.

    Prototypes are typically medium- to low-fidelity in terms of design elements (color scheme, typeface, images, and other visual assets). The goal of prototyping is to demonstrate how the site will function and what interaction will look like.

    There are some elements that prototyping tools such as Adobe XD, which is what I've used here, cannot do. One of those things is to search the site. There is a search button on the site to represent where that function will exist on the live site, but it is not operational on the prototype.

    Open Prototype in Adobe.com

  7. Prototype Testing Materials

    8. With prototypes created, and with stakeholder buy-in, the next step is to conduct prototype testing materials, which will be used to conduct site usability testing with representative site visitors.

    Open Testing Materials in Google Docs

  8. Usability Research Results and Recommendations

    9. With the prototype testing complete, it was time to summarize all the research results and recommendations. View the full presentation on YouTube or view the slides.

    View YouTube Presentation View Google Slides

    10. Further research has led to iterative changes. The overall color scheme and site layout have been updated to better align with current trends and usability standards. Visit the live site to see how those changes have translated.
