Autumallyear Recipe Website

Accessible, Engaging, and Performance-Optimized for Home Cooks

Autumnallyear.us connects food enthusiasts with easy-to-follow recipes, visually appealing layouts, and clear navigation. The redesign focused on accessibility, page performance, and user experience, ensuring visitors can quickly find, follow, and share recipes.

Work With Me

About Autumallyear.us

Autumallyear.us is a seasonal recipe website designed to showcase fresh ingredients, cooking guides, and curated seasonal dishes. The project involved a **ground-up build**, focusing on performance, accessibility, and a structure that allows easy content management and scalability.

The goal was to create a site where users could easily browse recipes, search by categories, filter by seasonal ingredients, and share content—all with minimal friction and optimal page speed.

Approach & Workflow

1. Discovery & Planning

  • Defined goals, audience, and key features for a recipe-focused website.
  • Planned content hierarchy, recipe categories, and metadata for SEO optimization.
  • Mapped user paths: browsing, searching, filtering, and sharing recipes.

2. Design

  • Created wireframes and high-fidelity UI reflecting seasonal aesthetics and readability.
  • Designed responsive layouts for mobile, tablet, and desktop devices.
  • Developed consistent typography, color palette, and interactive elements.

3. Content Strategy

  • Organized recipes into logical categories with clear titles and descriptions.
  • Implemented metadata, tags, and structured data for SEO and searchability.
  • Prepared imagery optimized for web performance without sacrificing quality.

4. Development

  • Built semantic HTML structure and modular CSS for maintainable styling.
  • Optimized images, lazy-loaded media, and minimized JS for fast performance.
  • Implemented accessibility best practices (ARIA roles, keyboard navigation, proper contrast).

5. Testing & Launch

  • Cross-browser and cross-device testing for consistent experience.
  • Accessibility audit to ensure compliance with WCAG 2.1 standards.
  • Performance testing with PageSpeed Insights and optimization of Core Web Vitals.

6. Post-Launch Optimization

  • Monitor performance metrics and make iterative improvements.
  • Update content, recipes, and images while maintaining performance.
  • Continually enhance SEO, structured data, and accessibility compliance.

These results show that prioritizing real-world usability, accessibility, and speed produces a high-performing site for all users—even on lower-end devices and slower networks. Lighthouse scores guide optimizations but are not the only measure of success.

Technical Highlights

  • Responsive layouts for all devices with CSS Grid & Flexbox.
  • Optimized images with explicit width/height and lazy-loading for fast performance.
  • Semantic HTML structure and ARIA attributes for accessibility.
  • Structured data markup for recipe metadata (JSON-LD) to improve SEO.
  • Performance monitoring via Google PageSpeed Insights and Core Web Vitals.

Key Learnings

Building Autumallyear.us from scratch allowed complete control over performance, accessibility, and content structure. Prioritizing usability and SEO from day one ensures a scalable, user-friendly website that can grow with future content and features.
